1. CLI Commands

All commands available from the pawbot binary.

Core

pawbot onboard Create config + workspace. Use --setup for interactive API key wizard.
pawbot agent Interactive chat session. Use -m "..." for single message.
pawbot gateway Start the gateway server (enables all channels). Default port: 18790.
pawbot status Show config, workspace, model, and provider status.

Channels

pawbot channels status Show connection status of all configured channels.
pawbot channels login Link WhatsApp via QR code scanning.

Providers

pawbot provider login <name> OAuth login for OAuth-based providers (e.g., openai-codex, github-copilot).

Cron (scheduling)

pawbot cron list List scheduled jobs. Use -a to include disabled jobs.
pawbot cron add Add a job. Requires --name, --message, and one of --every, --cron, --at.
pawbot cron remove <id> Remove a job by ID.
pawbot cron enable <id> Enable a job. Use --disable to disable.
pawbot cron run <id> Manually run a job now. Use -f to force disabled jobs.

4. Built-in Tools

Tools the agent can invoke during conversations.

Tool Source file Description
filesystem agent/tools/filesystem.py Read, write, list, search files in workspace
shell agent/tools/shell.py Execute shell commands with timeout and path control
web agent/tools/web.py Brave web search, URL reading
mcp agent/tools/mcp.py MCP server tool proxy (stdio + HTTP)
cron agent/tools/cron.py Schedule, list, remove cron jobs
message agent/tools/message.py Send messages to channels
spawn agent/tools/spawn.py Spawn subagent with budget

Tools config lives under tools in config.json. MCP servers are registered under tools.mcpServers.

6. Configuration Schema

All config lives in ~/.pawbot/config.json. Supports both camelCase and snake_case keys.

Root sections

Key Type Description
agents AgentsConfig Default model, provider, tokens, temperature, iterations
providers ProvidersConfig API keys and base URLs for each provider
channels ChannelsConfig Enable/disable channels, tokens, allowFrom lists
gateway GatewayConfig Server host, port, heartbeat interval
tools ToolsConfig Web search, shell exec, workspace restriction, MCP servers
heartbeat HeartbeatEngineConfig Proactive trigger engine (check interval, triggers file)
cron CronConfig Scheduler enabled, registry path, check interval
skills SkillsConfig Skill system toggle, auto-create, skills directory
lora LoRAConfig Fine-tuning (enabled, auto-train, base model, dataset)
security SecurityConfig Sandboxing, injection detection, audit log, salience
observability ObservabilityConfig Tracing, OTLP, Prometheus port, privacy controls
subagents SubagentsConfig Max concurrent, token/time budgets, inbox review

Agent defaults

Key Default Description
model anthropic/claude-opus-4-5 Default model to use
provider auto Auto-detect provider from model name
maxTokens 8192 Max output tokens
temperature 0.1 Sampling temperature
maxToolIterations 40 Max tool call rounds per conversation turn
memoryWindow 100 Number of recent messages to include in context
reasoningEffort null low / medium / high — enables LLM thinking mode

Security defaults

Key Default Description
enabled true Security layer enabled
requireConfirmationForDangerous true Prompt before dangerous commands
blockRootExecution true Prevent running as root
injectionDetection true Detect prompt injection attempts
minMemorySalience 0.2 Minimum salience to store a memory
maxMemoryTokens 300 Max tokens per memory entry
auditLogPath ~/.pawbot/logs/security_audit.jsonl Path to audit log

7. Architecture

Pawbot's core modules:

Module Files Purpose
Agent Loop agent/loop.py Core conversation loop with tool calling, complexity classification, pre/post-task reflection
Context Builder agent/context.py Budget-aware context assembly, relevance-based loading, prompt cache markers
Memory agent/memory.py SQLite fact store, vector embeddings, RAG retrieval, salience decay
Security agent/security.py Command sandboxing, injection detection, audit logging
Skills agent/skills.py Skill loading, execution, auto-creation from novel tasks
Subagents agent/subagent.py Parallel background execution pool with budget and concurrency control
Telemetry agent/telemetry.py OpenTelemetry spans, Prometheus metrics, trace export
Providers providers/ LiteLLM routing, provider registry, model matching, OAuth flows
Channels channels/ 10 channel adapters with unified message bus interface
Message Bus bus/queue.py Async message routing between channels, agent, and subagents
Cron cron/ Job scheduling with cron expressions, intervals, one-time runs
Heartbeat heartbeat/ Proactive trigger engine with condition-based wake-ups
Config config/ Pydantic schema, JSON loader, migration, env var support

Data directory

~/.pawbot/ ├── config.json # Main configuration ├── workspace/ # Agent workspace │ ├── SOUL.md # Agent identity │ ├── USER.md # User profile │ ├── TOOLS.md # Available tools │ ├── memory/ │ │ ├── MEMORY.md # Long-term memory │ │ └── HISTORY.md # Conversation history │ └── skills/ # User-created skills ├── cron/ │ └── jobs.json # Scheduled jobs └── logs/ ├── traces.jsonl # OpenTelemetry traces └── security_audit.jsonl

8. Environment Variables

All config fields can be set via env vars with prefix PAWBOT_ and double-underscore nesting:

# Equivalent to config.providers.openrouter.apiKey export PAWBOT_PROVIDERS__OPENROUTER__API_KEY="sk-or-v1-..." # Equivalent to config.agents.defaults.model export PAWBOT_AGENTS__DEFAULTS__MODEL="anthropic/claude-sonnet-4-5" # Equivalent to config.tools.web.search.apiKey export PAWBOT_TOOLS__WEB__SEARCH__API_KEY="BSA-..."

Env vars take precedence over config.json values. The config class uses pydantic-settings with env_prefix="PAWBOT_" and env_nested_delimiter="__".
